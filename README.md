## 配信前に必須の設定

`lp/a1d1/index.html` 冒頭の `TODO(必須)` を検索すること。

1. **`FORM_ENDPOINT`** — Power Automate「HTTP要求の受信時」のトリガーURL。
   **未設定の間はリードが1件も届かない**（送信は中止され、偽の完了画面は出さない仕様）。
   フロー設計は hikari-hq の `docs/power-automate-lead-flow.md` を参照。
2. **運営者表示・プライバシーポリシー** — フッターが未記載。
   特定商取引法・個人情報保護法の表示義務があり、未記載では広告審査に通らない。

## このリポジトリを触る際の注意

- **公開リポジトリ**。広告予算・CPA目標・撤退基準などの戦略情報を書かないこと。
- `FORM_ENDPOINT` のURLは SAS署名を含み、**公開される**。これは Power Automate の仕様で回避不能。
  悪用時は SASキーを再生成し、ここのURLを差し替えて再デプロイする（commit1本で復旧できる）。
- エリア判定は**都道府県レベルの陰性フィルタ**。「提供可能」の断定はしていない。
  理由は `lp/a1d1/index.html` のエリア判定ブロック冒頭のコメントに全て書いてある。**文言を変える前に必ず読むこと。**
